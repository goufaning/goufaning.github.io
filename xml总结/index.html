什么是xml

XML英文名字为：Extensible Markup Language ，是可扩展标记语言，是在HTML、SGML等的基础发展而来的语言，它很像HTML（超文本标记语言），但其设计宗旨是传输数据，而不是显示数据。

xml与html的区别

Html xml
可扩展性 不具有扩展性 可自行定义元素
作用 描述数据的性质和结构 包含和传输数据
语法要求 不区分大小写，可以有多个根元素，属性值的引号可用可不用，空格会自动过滤等 严格区分大小写，只能有一个根元素，属性值必须放到引号中，空格不会自动删除等
数据与格式 数据与格式在一起 数据与格式分离

需要注意的是，XML不是HTML的升级，也不是HTML的替代产品。

  1. 文档编码声明

编码声明标识用于表示文档中的字符的编码。 尽管 XML 分析器可以自动确定文档使用的是 UTF-8 还是 UTF-16 Unicode 编码，但是，在支持其他编码的文档中应使用此声明。

  1. 独立文档信息

独立声明指示文档的内容是否依赖来自外部源的信息，例如外部文档类型定义 (DTD)。，如果文档依赖于外部文档，则设standalone=”no”来声明。默认情况下为no。

注意：如果同时设置了encoding和standalone属性，standalone属性要位于encoding属性之后。

xml约束

在XML技术里，可以编写一个文档来约束一个XML文档的书写规范，这称之为XML约束。XML文档的约束可以规定该文档都有哪些标签，包括标签的属性等等，都可以在约束中进行规范编写。

元素定义

声明一个XML元素

元素类型描述主要有如下几种：

类型 描述 语法
任意类型 这种元素即可以是字符串，也可以是其他子元素
字符串值 这种元素只能是字符串，不能包含其他子元素
空元素 这种元素只能为空元素，空元素前面有介绍
包含子元素 元素中可以出现子元素
混合类型 可以出现以上的类型

子元素

1.定义有序的子元素：如果使用引文英文逗号（,）作为子元素之间的分隔符，则子元素之间必须遵守定义的顺序。
2.定义互斥的子元素：互斥的子元素表名一系列子元素之间只能出现其中的一个，互斥的子元素用竖线（|）分隔.
3.子元素出现的频率：
| 符号 | 表示频率 |
| —- | —————————– |
| + | 表明子元素可以出现1次或者多次 |
| * | 表明子元素可以出现0次或者多次 |
| ? | 表明子元素可以出现0次或者1次 |

常用属性值类型

类型 说明
CDATA 表示属性值为普通文本字符串
ENUMERATED 该属性值是一系列枚举值之一(en1丨en2丨en3)
ID 该属性值必须是有效的标示符，且该属性值可用于表示该元素，在XML中必须唯一
IDREF 该属性值必须引用自另一个已有的ID属性值
IDREFS 该属性值必须引用自多个已有的ID属性值，多个ID属性值之间用空格隔开
NMTOKEN 该属性值必须是一个合法的XML名称，它也指定了该属性值是字符串数据，但比CDATA具有更强的约束，它表明属性值只能由字母、数字、英文下划线、英文中划线、英文点号、英文冒号等组成
NMTOKENS 该属性值可以是多个NMTOKEN，过个NMTOKEN之间用空格隔开
NOTATION 该属性值是在DTD声明过的符号（NOTATION），这是个将要过期的规范看，避免使用
ENTITY 该属性值是一个外部实体，例如图片文件
ENTITIES 该属性值是多个外部实体，多个实体之间用空格隔开
